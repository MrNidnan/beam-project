# User Manual: Daily Use

Once Beam is configured, daily use should be simple.

## Typical Workflow

A normal Beam session looks like this:

1. Start Beam.
2. Start your music player.
3. Confirm the current song appears in the Beam preview.
4. Open or show the display window.
5. Start the event.

## During the Event

You usually do not need to keep changing settings while Beam is running.

Most users only do these things during an event:

- check the preview
- change mood or background if needed
- hide or show the display window
- move the display to the correct screen if Windows changes monitor order

## When You Pause Playback

Beam can keep the last song visible while playback is paused.

This is useful during announcements or short pauses.

## When Playback Stops

When playback stops, Beam returns to the default or not-playing mood.

That keeps the screen clean between tandas or at the end of the event.

## Before You Close Beam

If you changed settings, click `Apply` so Beam saves them.
